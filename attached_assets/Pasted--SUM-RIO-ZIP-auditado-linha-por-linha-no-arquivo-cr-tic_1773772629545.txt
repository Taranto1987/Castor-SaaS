## ARTEFATOS

### 1. CORREÇÃO CRÍTICA — UPSERT REAL

Substituir:

```ts
await db.insert(produtosTable).values({...});
```

POR:

```ts
await db.insert(produtosTable)
.values({
  nome: item.name,
  sku: item.sku,
  preco,
  precoPix,
  parcelamento: `12x de ${formatBRL(precoFinal / 12)}`,
  medidas: medidas || null,
  altura: altura || null,
  categoria: categoria.nome,
  imagem: imagem || null,
  link,
})
.onConflictDoUpdate({
  target: produtosTable.sku,
  set: {
    preco,
    precoPix,
    parcelamento: `12x de ${formatBRL(precoFinal / 12)}`,
    imagem,
    link
  }
});
```

---

### 2. CORREÇÃO DE DUPLICIDADE (JÁ EXISTE NO CÓDIGO)

Você já tem:

```ts
const seenIds = new Set<number>();
```

Mas isso NÃO protege o banco.

→ precisa ser reforçado com:

```ts
if (!item.sku) continue;
```

---

### 3. GARANTIA DE CONSISTÊNCIA

Adicionar antes do loop:

```ts
await db.execute(`
  CREATE UNIQUE INDEX IF NOT EXISTS produtos_sku_unique ON produtos (sku);
`);
```

---

### 4. CORREÇÃO DE CONTAGEM (ERRO DE ILUSÃO)

Hoje:

```ts
totalEstimado += total_count
```

Problema:

* produtos repetidos em múltiplas categorias

Correção:

```ts
const totalEstimado = new Set();
```

e alimentar com `item.id`
